# AGENTS.md
## レビューについて
ヘッダファイルおよび verify ソースコードだけでなく、ドキュメントもレビューすること。

特に、以下の点を必ず確認すること。

### 共通
- 変更するファイルの末尾に改行がある。
- 句読点が基本的に `、`, `。` である（`,`, `.` などでない）。
  - ただし、`$0$, $1$, $2$` などのように英数字を並べる際のカンマまで全角に修正させる必要はない（どちらでもよい）。
- マークダウンの箇条書きが `- ` である（`* ` や `  - ` などでない）。
  - マークダウンの箇条書きのインデントがスペース 2 文字である。

### ドキュメント
- 数式として書くべきである制約などの記述は、バッククォートによる囲みなどではなく TeX 数式として書いてある。
  - 引数などの複数文字で表される値が数式内に現れる場合は、基本的に `\mathrm{ }` で記述している。
  - さらに、積をとるなど `\mathrm{ }` そのままで記述すると見にくい場合は適切に他の文字でおき、文字でおいたことも記述している。
- TeX 数式の囲みが `$ $` である（`\( \)` などでない）。
- TeX 数式での絶対値記号による囲みが `\lvert \rvert` である（`| |` などでない）。

### verify ソースコード
- オンラインジャッジ上の問題を使いつつ自己検証も行う verify ソースコードについて、自己検証を行う関数の関数名が `self_test` であり、呼び出す際は `#ifndef ONLINE_JUDGE`, `#endif` で囲んでいる。
- AOJ の URL は `https://judge.u-aizu.ac.jp/onlinejudge/description.jsp?id=...` の形式である（`http://...` などでない）。

# CONTRIBUTING.md

NicheLibrary への貢献をご検討していただきありがとうございます。  
このライブラリは個人で管理するニッチなものであり、方針と整合しない変更は採用しないことがあります。

## 言語

- ドキュメント・ソースコード中のコメントは**日本語**で書くようにお願いします。
- Issue / Pull Request の description は日本語以外（英語など）でも受け付けます。

## 受け付けるもの / 受け付けないもの

### ✅ 歓迎: 既存の実装の不具合の報告・修正

- 不具合を報告する Issue
  - 再現条件、期待される挙動、不具合が発生する小さいケース（可能であれば）など
- 不具合を修正する PR（verify の追加・修正を含む）
  - レビューで細かい修正をお願いする可能性があります。

### △ 条件付きで受け入れる可能性: 新規の実装を追加する PR

以下を全て満たすような場合には、受け入れる可能性があります。

- [Luzhiled's Library](https://ei1333.github.io/library/) / [Nyaan's Library](https://nyaannyaan.github.io/library/) / [cp-library-cpp](https://suisen-cp.github.io/cp-library-cpp/) の全てに代用できる実装が存在しない。
  - 「同じデータ構造・アルゴリズムの実装はあるが、一部のケースに対応できない」「公開されている関数が使いづらい」などは代用できないとしてよい。
- competitive-verifier で **verify できる既存の問題**がある。
  - yukicoder または Library Checker を推奨。その 2 つにはないが Aizu Online Judge にある場合は AOJ でもよい。
- 実装やドキュメントが本リポジトリの既存の形式に沿っている。
  - 必要に応じて私がレビューで修正を依頼する。

### ❌ 基本的に対応しない: 追加の要望の Issue

- 追加の要望の Issue には基本的に応じません。
- ただし、上記の 3 ライブラリに無く、かつ私自身も使う可能性があると判断したものは、クローズせず将来的に対応する可能性があります（対応時期は未定です）。

## 必須: verify の追加と維持

- 新規の実装を追加する場合は、verify 用のソースコードを追加してください。
- 既存の実装の修正でも、必要に応じてテストの追加や更新をお願いします。

## 必須: コーディング規約（GCC / Clang 両対応）

以下を満たすようにしてください。

- GCC / Clang の両方でコンパイルができ、警告が出ない。
  - コンパイルオプションとして、 `-std=c++20` は仮定してよい。
- `#include <bits/stdc++.h>` を使用しない。
- `using namespace std;` を使用しない。
- 使っていない標準ライブラリヘッダを include しない（必要最小限にする）。

## 必須: clang-format-20 によるフォーマット

- リポジトリの `.clang-format` に従い、**clang-format-20** でフォーマットしてください。
- フォーマットされていない場合、CI が通らずにマージできないように設定されています。
- clang-format のバージョンが古いと、同じ `.clang-format` でも結果が異なる場合があります。

## PR 作成前チェックリスト

- [ ] verify の追加と維持
- [ ] clang-format-20 によるフォーマット
- [ ] GCC / Clang の両方でコンパイルができ、警告が出ない
- [ ] ドキュメントの日本語での追加・更新（必要であれば）
